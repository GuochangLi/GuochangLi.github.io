搜索

分类

Type1

无信息搜索

  • 所有信息都已经给出，没有额外的信息

有信息搜索

  • 除了给出的信息，还有额外的信息
  • 额外的信息可以提高搜索效率

Type2

确定的，全观测

  • 确定：在某个状态下，能够确定下一个状态
  • 能够看到所有信息(可枚举全部信息)

不确定，不全观测(现在不涉及)

  • 不确定：在某个状态下，无法判断下一个状态
  • 不全观测：能够看到部分信息或者不能看到任何信息

问题描述

  • 状态
  • action：根据一定的策略将要采取的行为
  • solution：搜索的结果

Example

  • 一些假设

    • 状态：每个城市
    • action：从当前城市出发到达相邻的城市
    • 转移模型：在当前的城市采取action到达的城市
    • Path
    • 目标测试：测试最后到达的城市是否是我们要求的城市
    • Path cost：这个题目中Path中所有距离和

搜索算法

构造

  • 从初始状态到结束状态能够构建一个搜索树
  • 一个节点的孩子是这个状态可能到达的下一个节点
  • 叶子节点：没有后续状态
  • 边缘节点(frontier)：即将被拓展的节点
  • 构造过程：一个节点出发，将可拓展的节点放入frontier集合中，从frontier中选取下一个可以被拓展的节点重复之前的操作直到找到结果或者frontier集合为空则结束
  • 一些问题：
    • 循环路径

一些比较

有信息搜索策略

贪婪优先

  • 启发函数h(n)：估计一个节点到达目标节点的最短路径或者最小开销
  • 估值函数f(n)：估计n是否能够从frontier中选出作为下一个最好的后续状态
  • 贪婪算法中f(n)也用h(n)表示
问题描述
  • 额外信息：任意一个节点到达目标城市的直线距离
  • 启发函数的值就是这个城市到达目标城市的直线距离
算法描述
  • 从frontier中选取的时候要根据h(n)来选取h(n)最小的节点
评价
  • 可能不是完备
  • 可能不是最优

A*

  • Goal：解决贪婪算法不是完备的问题
问题描述

  • h(n)：同贪婪优先中的定义相同

  • f(n)=g(n)+h(n)：g(n)从初始节点到达目标节点的实际开销

评价
  • 最优：YES
    • 可容性(针对启发函数)：启发函数不过高估计
    • 一致性：单调递增（比一致性更强的性质，满足一致性一定满足可容性）
  • 其他启发函数
    • 汉明距离
    • 曼哈顿距离

对抗搜索

  • 在寻找最优结果的时候，有阻碍
  • 博弈相关

问题描述

  • 起始状态(S0)
  • 玩家：哪一个玩家
  • actions：玩家s施加的行为
  • result(s, a)：s施加a得到的结果
  • Terminal-Test：是否停止
  • Utility：游戏结束，位于当前状态的收益

蒙特卡洛树搜索

  • 有选择性的随机采样

蒙特卡洛方法

描述

  • 通过不断重复随机采样，用随机性解决问题
  • keypoint：采样

原理

  • 在一个问题中，只要采样足够多样本均值等于期望一致
  • 期望：所有样本的平均值

蒙特卡洛仿真

  • 学习的时候是学习了policy

  • 探索与利用已有的数据，得到新的policy

  • 已有数据与探索性的平衡

蒙特卡洛规划

Selection

  • 根节点出发，选择后续孩子节点，选择策略(探索与利用结合)

Expansion

  • 孩子节点有违背访问节点，则拓展这个节点

Simulation

  • 找到仿真过的叶子节点

Backpropagation

  • 仿真结束，回溯更新信息(游戏次数，获胜次数)
